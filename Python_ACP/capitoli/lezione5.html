5  Passaggio parametri, *args, **kwargs

Tutto in Python è un oggetto. Possiamo interagire con gli oggetti attraverso dei riferimenti.

  • Quando si dichiara una variabile, questa non contiene effettivamente il valore, ma piuttosto la sua posizione in memoria.
  • È possibile utilizzare la funzione id(...) per conoscere il riferimento.
x = 7
y = 10
print("x=", x, "ref_x=", id(x))
print("y=", y, "ref_y=", id(y))
x= 7 ref_x= 140228276227184
y= 10 ref_y= 140228276227280

5.1 Passaggio di parametri

Tutti i parametri in Python sono passati per riferimento.

Quando passiamo un parametro ad una funzione, il riferimento contenuto nel parametro attuale è copiato all’interno del parametro formale.

Utilizzare tale riferimento significa avere accesso direttamente all’oggetto in memoria.

def fun(x):
    print("Inside fun: x=", x, "ref_x=", id(x))
x = 7
fun(x)
print("Outside fun: x=", x, "ref_x=", id(x)) 
Inside fun: x= 7 ref_x= 140228276227184
Outside fun: x= 7 ref_x= 140228276227184

Nel caso in cui il parametro fosse un oggetto mutabile, ogni modifica al parametro all’interno della funzione, si riflette anche sul chiamante.

def fun(mylist):
    mylist.append([1,2,3,4])
    print ("Inside fun: val=",  mylist, "ref=", id(mylist))
mylist = [10,20,30]
fun( mylist )
print ("Outside fun: val=", mylist, "ref=", id(mylist))
Inside fun: val= [10, 20, 30, [1, 2, 3, 4]] ref= 140227671125312
Outside fun: val= [10, 20, 30, [1, 2, 3, 4]] ref= 140227671125312

Ovviamente questa modifica si rifletterà a meno che il riferimento contenuto del parametro formale della funzione non venga sovrascritto. In tal caso il parametro diventa locale alla funzione.

Pertanto, ogni modifica all’interno della funzione, non si riflette sul chiamante.

def fun(mylist):
    mylist = [1,2,3,4]
    print ("Inside fun: val=",  mylist, "ref=", id(mylist))
mylist = [10,20,30]
fun( mylist )
print ("Outside fun: val=", mylist, "ref=", id(mylist))
Inside fun: val= [1, 2, 3, 4] ref= 140227671212608
Outside fun: val= [10, 20, 30] ref= 140227671123200

Nel caso in cui il parametro sia un oggetto immutabile, anche se la funzione dispone del riferimento all’oggetto, non gli sarà possibile apportare modifiche che si riflettano nel chiamante.

Perché le modifiche che potremmo fare sono quelle in-place che significano creare un nuovo oggetto.

5.2 Argomenti a lunghezza variabile: *args

La sintassi *args può essere utilizzata nella definizione della funzione Python per passare alla funzione un numero variabile di argomenti.

La funzione riceverà una tupla di argomenti, la quale può essere iterate per recuperare tutti gli argomenti.

def fun(*args):
    for arg in args:
        print(arg)


fun()

fun("giovanni",22,1,["matteo", "caspita"])
giovanni
22
1
['matteo', 'caspita']

5.3 Keyword arguments

I keywords arguments permettono al chiamante di una funzione di passare alla funzione gli argomenti basandosi sul nome, e non sulla posizione.

La sintassi da utilizzare è del tipo key = value

def fun(name, surname, country):
    print(name,surname,country)

fun(country="italy", name="giovanni", surname="fragola")
giovanni fragola italy

5.4 **kwargs

La sintassi **kwargs può essere utilizzata nella definizione delle funzioni Python per passare alla funzione un numero variabile di keyword arguments.

La funzione riceverà un dizionario di argomenti (kwargs nell’esempio), il quale può essere iterato per recuperare tutti gli argomenti.

def fun(**kwargs):
    for key, value in kwargs.items():
        print(f"{key:<10}{value}")

fun(day=8, month=5, year=2006)

fun(name="giovanni", surname="fragola", email="giovanni.fragola.fragola")

day       8
month     5
year      2006
name      giovanni
surname   fragola
email     giovanni.fragola.fragola

5.5 Argomenti di default

Python permette di definire dei valori di default per gli argomenti di una funzione.

I valori di default di un argomento sono utilizzati quando nella chiamata alla funzione non è specificato un valore per quell’argomento.

Questo permette di invocare una funzione con meno parametri rispetta alla quantità presente nella sua definizione.

def fun(username, password="changename", days=20):
    print(username, password, days)

fun("giovanni", "password", 90)

fun("giovanni")
giovanni password 90
giovanni changename 20

Gli argomenti con parametri di default devono essere specificati dopo quelli obbligatori.

# ERORRE
def fun(username, password="prova", days):
    print(days, username, password)
  Cell In[22], line 2
    def fun(username, password="prova", days):
                                        ^
SyntaxError: parameter without a default follows parameter with a default

5.6 Unpacking argument list

La sintassi *args utilizzata per i variable-lenght arguments può essere anche utilizzata per operazioni di unpacking degli argomenti.

  • Gli argomenti sono già inseriti in una lista o tupla ma devono essere utilizzati in una funzione che richiede argomenti utilizzati in maniera separata.
l = list(range(3,7))
print(l)
arg = [3,7]
l = list(range(*arg))
print(l)

t = tuple(range(3,7))
print(t)
arg = (3,7)
t = tuple(range(*arg))
print(t)
[3, 4, 5, 6]
[3, 4, 5, 6]
(3, 4, 5, 6)
(3, 4, 5, 6)

6 Gestione dei File e delle Eccezioni in Python

Quando eseguiamo un programma Python creiamo 3 oggetti file standard:

  • sys.stdin
  • sys.stdout
  • sys.stderr
with open("accounts.txt", mode="w") as accounts:
    accounts.write("100 Jones\n")
    accounts.write("99 Anna\n")
    accounts.write("88 Luigi\n")
    accounts.write("11 Matteo\n")

6.1 L’istruzione with

Le applicazioni acquisiscono risorse e dovremmo rilasciare le risorse non appena non sono più necessarie (sono limitate).

with quando viene utilizzata per acquisire una risorsa e utilizzarla.

Alla fine dello scope di with viene chiamato implicitamente il metodo close dell’oggetto risorsa per rilasciare la risorsa.

6.2 open e write

La funzione open apre un file e lo associa ad un oggetto file.

L’argomento mode specifica la modalità di apertura del file, indicando se aprire un file per la lettura del file, per la scrittura nel file o per entrambi.

Con la modalità w, Python crea il file nella cartella corrente se questo non esiste.

Durante la lettura dal file è possibile iterare sull’oggetto file grazie al costrutto for

for line in file

Dove leggiamo una riga alla volta del file e la restituisce come stringa.

6.3 readlines e seek

Esiste anche il metodo readlines() che può essere utilizzato per leggere un intero file di testo.

Il metodo restituisce ogni riga del file come una stringa in una lista di stringhe

ATTENZIONE: Utilizzare readlines per file di grandi dimensioni può essere un’operazione che richiede molto tempo, e in tale tempo non possiamo utilizzare la lista di stringhe che viene restituita.

Il metodo seek() ci permette di posizionare il puntatore alla riga corrente.

6.4 Modulo os per la gestione file

Il modulo os in Python consente di utilizzare funzionalità dipendenti dal sistema operativo in maniera semplice.

Per la gestione dei file possiamo utilizzare le funzioni di rimozione e rinomina.

import os

os.remove("...")
os.rename("file.txt", "FILE.txt")

6.5 Formattazione dell’output

Finora abbiamo visto 3 modi per scrivere valori su file

  • Dichiarazioni di espressione
  • Funzione print()
  • Uso del metodo write() sugli oggetti

Per avere un controllo maggiore sulla formattazione dell’output rispetto alla semplice stampa di valori separati da spazi esistono diversi modi in Python.

È possibile utilizzare i cosiddetti formatted string literals. Per utilizzarli:

  • Iniziare una stringa con f o F prima del single quote o double quote
  • All’interno di questa stringa, si può scrivere un’espressione Python tra i caratteri {expression} che può fare riferimento a variabili o a valori letterali.
table = {'jack': 435, 'Gigi': 3645, 'giovanni': 55}

for name, number in table.items():
    print(f"{name:10} ==> {number:10d}")
jack       ==>        435
Gigi       ==>       3645
giovanni   ==>         55

Passando un intero dopo :, il campo sarà largo un numero minimo di caratteri. Questo è utile per allineare le colonne
